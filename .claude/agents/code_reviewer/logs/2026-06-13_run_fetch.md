## 1. Script Summary

`_run_fetch.py` fetches a fixed list of six CSV files from a public GitHub raw URL set using `httpx` in async mode, then writes each successfully downloaded file to a timestamped local data directory. It also configures a timestamped log directory that captures both console and file output. The script is designed to be run as a one-shot entry-point but currently lacks an `if __name__ == "__main__":` guard, causing all setup logic to run on import.

---

## 2. Findings

### Critical

**C-1: TLS/SSL verification is disabled (`verify=False`) — Line 52**

`httpx.AsyncClient(verify=False)` disables certificate validation entirely, exposing every download to man-in-the-middle attacks. The target URLs are plain public HTTPS endpoints on `raw.githubusercontent.com`, which serves a valid certificate, so there is no legitimate reason for this.

Fix: Remove the `verify` argument (default is `True`).
```python
# Before
async with httpx.AsyncClient(verify=False) as client:
# After
async with httpx.AsyncClient() as client:
```

---

### Major

**M-1: Module-level side effects execute on import — Lines 20–35**

`timestamp`, `data_dir`, `log_dir`, directory creation (`mkdir`), and `logging.basicConfig` all run at module scope. Importing this file in a test or notebook immediately creates directories and reconfigures the global logger. Fix: move all setup into `main()` or a helper function and add `if __name__ == "__main__":`.

**M-2: Bare `except Exception` swallows all error types — Lines 45–47**

The project standard states "catch only what you can handle." Catching bare `Exception` hides programming errors (`TypeError`, `AttributeError`, `asyncio.CancelledError`). Fix: catch only httpx-specific exceptions (`httpx.TimeoutException`, `httpx.HTTPStatusError`, `httpx.RequestError`).

**M-3: No module-level or function-level docstrings — Lines 1, 38, 50**

Neither the module nor `fetch_one` nor `main` has a docstring. CLAUDE.md mandates PEP 257 docstrings for all public functions, classes, and modules.

---

### Minor

**m-1: Module-level constants lack type annotations — Lines 10–25**
`URLS`, `BASE`, `timestamp`, `data_dir`, `log_dir` have no type hints. Fix: annotate as `URLS: list[str] = [...]`, `BASE: Path = Path(__file__).parent`.

**m-2: `log` is a non-descriptive variable name — Line 35**
Convention is `logger`. Fix: `logger: logging.Logger = logging.getLogger(__name__)`.

**m-3: `r` is a non-descriptive variable name — Line 41**
Single-letter names reduce readability. Fix: rename to `response`.

**m-4: `csv` and `io` imports are unused — Lines 3–4**
Both are imported but never used. PEP 8 / F401 violation. Fix: remove both import lines.

**m-5: Magic number `30` for timeout — Line 41**
Fix: `REQUEST_TIMEOUT_SECONDS: int = 30` at module level, then reference it.

---

### Suggestions

**S-1:** Load URLS from a config file rather than hardcoding them in source.
**S-2:** Collect and log the filenames of failed downloads, not just the count.
**S-3:** Set `follow_redirects=True` on the client constructor rather than repeating it per-request.
**S-4:** Add `encoding="utf-8"` to the `FileHandler` — the current log has garbled characters (`?`) for em-dashes on Windows.

## 3. Top 3 Priority Fixes

### Priority 1 — Re-enable TLS verification (C-1)
```python
# Before
async with httpx.AsyncClient(verify=False) as client:
# After
async with httpx.AsyncClient() as client:
```
This is the only security vulnerability — one-word change, highest impact.

### Priority 2 — Narrow exception catch (M-2)
```python
# Before
except Exception as exc:
    log.error("FAILED   %s  error=%s", url, exc)
    return filename, None, str(exc)

# After
except httpx.TimeoutException as exc:
    log.error("TIMEOUT  %s  error=%s", url, exc)
    return filename, None, str(exc)
except httpx.HTTPStatusError as exc:
    log.error("HTTP ERR %s  status=%d", url, exc.response.status_code)
    return filename, None, str(exc)
except httpx.RequestError as exc:
    log.error("REQUEST FAILED  %s  error=%s", url, exc)
    return filename, None, str(exc)
```

### Priority 3 — Move setup into functions, guard with `__main__` (M-1)
```python
def _setup_environment() -> tuple[Path, Path]:
    """Create timestamped data/log directories and configure logging."""
    timestamp = datetime.now().strftime("%Y-%m-%d_%H-%M-%S")
    base = Path(__file__).parent
    data_dir = base / "data" / timestamp
    log_dir = base / "logs" / timestamp

    logging.basicConfig(
        level=logging.INFO,
        format="%(asctime)s %(levelname)s %(message)s",
        handlers=[
            logging.FileHandler(log_dir / "fetchAPI.log", encoding="utf-8"),
            logging.StreamHandler(),
        ],
    )
    return data_dir, log_dir

async def main() -> None:
    """Orchestrate async download of all URLS and persist results to disk."""
    data_dir, log_dir = _setup_environment()
    ...

if __name__ == "__main__":
    asyncio.run(main())
```
